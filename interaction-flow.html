Microinteraction: Open Review

Trigger
:Review, :Review main, or :ReviewToggle.
Feedback
Review tab opens with Files, Diff, and Threads surfaces.
Rules
GitButler workspaces use GitButler merge base; regular local sessions can change base later.
Modes
Local, explicit ref, PR/MR, GitButler.

Microinteraction: Choose Scope

Trigger
<Tab> cycles all/current/select commit modes.
Feedback
File list and thread queue filter to the active review unit.
Rules
Select-commit mode exposes commit/GitButler unit rows. Normal mode keeps the rail clean.
Modes
All changes, current commit, selected commit/unit.

Microinteraction: Change Base

Trigger
<leader>B or :ReviewChangeBase [ref].
Feedback
Prompt asks for ref when omitted; review re-diffs after selection.
Rules
Only local git review sessions. GitButler base is fixed to workspace merge base.
Modes
Prompt mode, direct command mode, baseline compare mode.

Microinteraction: Compare Units

Trigger
C from a unit row, or :ReviewCompareUnit [idx].
Feedback
Centered comparison popup shows overlap, left-only, and right-only file groups.
Rules
Requires at least two review units and a selected/current unit. If no target is provided, a picker opens.
Modes
Direct target from selected row, picker target, command target.

Microinteraction: Review Code

Trigger
<CR> opens file, ]c/[c move hunks, s toggles split in diff.
Feedback
Diff cursor stays on code; split view preserves scroll and returns from rails with s.
Rules
Visual selections and yanks exclude line numbers and diff signs.
Modes
Unified diff, split diff, visual line, visual block.

Microinteraction: Deliver Feedback

Trigger
N opens notes list; s stages; P publishes.
Feedback
Notes move through draft/staged/published states; clipboard commands confirm copy target.
Rules
GitButler notes publish only when a remote review target can be resolved.
Modes
Draft, staged, resolved, published, stale, local-only handoff.
Interaction-flow intent: each card documents the trigger, feedback, rules, and modes for a specific UI behavior, while the wireframes show where the user performs that action and what screen/popup appears next.
