1. INTRODUCTION                                          *review-introduction*

review.nvim is a minimal code review plugin for Neovim. It provides a
dual-panel layout for reviewing git diffs with inline notes.

Features:
- Dual-panel layout: file explorer + diff viewer
- Unified and side-by-side split diff views
- Word-level diff highlighting
- Commit-level navigation with per-commit file filtering
- Inline notes on diff lines (normal and visual range selection)
- Suggestion notes with GitHub-compatible `suggestion` blocks
- Notes list panel with draft/staged/published workflow
- Note persistence across sessions
- Reference other notes with `#<id>` syntax
- Export notes to markdown
- Colorblind-friendly color scheme (enabled by default)

==============================================================================
2. REQUIREMENTS                                          *review-requirements*

- Neovim >= 0.9.0
- git
- gh CLI (https://cli.github.com/) — for GitHub PR features
- glab CLI (https://gitlab.com/gitlab-org/cli) — for GitLab MR features
- plenary.nvim — for running tests

3. SETUP                                                      *review-setup*

Add the plugin with your package manager, then call setup: >lua

    require("review").setup()
<
All options have sensible defaults. See |review-configuration| for the full
list.

You can also set up a keymap to toggle the review panel: >lua

    vim.keymap.set("n", "<leader>rr", "<cmd>ReviewToggle<cr>")
<

4. COMMANDS                                                *review-commands*

                                                                    *:Review*
:Review [ref]           Open a diff review. If {ref} is given, diffs the
                        working tree against that ref (e.g. `HEAD~3`,
                        `main`, a commit SHA). If no ref is given, shows
                        unstaged changes.

                        When diffing against a ref that has multiple commits
                        in the range, the commit list is shown in the
                        explorer panel. Select a commit to filter files to
                        that commit's changes.

                        To review a PR/MR: check out the review branch first
                        (`gh pr checkout <number>` for GitHub or
                        `glab mr checkout <number>` for GitLab), then run
                        `:Review main` to see the diff.

                                                             *:ReviewToggle*
:ReviewToggle           Toggle the review panel. Opens if closed, closes
                        if open. Also available as: >lua

                            require("review").toggle()
<
                                                            *:ReviewHelp*
:ReviewHelp             Open a floating help window with review commands
                        and buffer-local keymaps.

                                                             *:ReviewNotes*
:ReviewNotes            Open the notes list for the current review session.

                                                           *:ReviewComment*
:ReviewComment [target] Add a review note. This behaves like pressing `a`
                        in the diff buffer, or you can pass an explicit
                        target:

                            :ReviewComment README.md:103
                            :ReviewComment lua/review.lua 42 old
<
                                                        *:ReviewSuggestion*
:ReviewSuggestion       Add a suggestion note. Works from the current
                        cursor/selection or with an explicit target like
                        |:ReviewComment|.

                                                              *:ReviewClose*
:ReviewClose            Close the review layout and end the session.

                                                             *:ReviewSubmit*
:ReviewSubmit           Submit all accumulated draft comments as a GitHub
                        review. Only available in PR mode. (Planned feature)

                                                             *:ReviewExport*
:ReviewExport [path]    Export all notes to a markdown file. Defaults to
                        `review-notes.md` in the current directory.

                                                          *:ReviewClipboard*
:ReviewClipboard        Copy all current notes and remote threads to the
                        clipboard for sharing with an LLM or another tool.
                        This includes local notes, open threads, and
                        discussion items, but skips resolved threads.

                                                     *:ReviewClipboardLocal*
:ReviewClipboardLocal   Copy only local draft/staged notes to the clipboard.

                                                         *:ReviewClearLocal*
:ReviewClearLocal       Clear all local draft/staged notes after confirmation.
                        Remote GitHub/GitLab threads are kept.

5. KEYMAPS                                                  *review-keymaps*

All keymaps are buffer-local to the review buffers and configurable via
|review-configuration|.

Explorer panel:~

    <CR>        Open the file or thread under the cursor. Thread entries
                jump to the related diff location.
    ?           Open review help.
    T           Cycle between full stack view and per-commit filtering.
    f           Focus the Files section.
    t           Focus the Threads section.
    q           Close the full review layout.

Diff viewer:~

    a           Add a note on the current line (also works in visual mode).
    S           Add a suggestion note (pre-populated with selected code).
                Works in normal and visual mode.
    e           Edit the note on the current line.
    d           Delete the note on the current line.
    N           Show all notes in a floating list.
    ?           Open review help.
    ]c          Jump to the next hunk.
    [c          Jump to the previous hunk.
    ]f          Jump to the next file.
    [f          Jump to the previous file.
    n / ]n      Jump to the next note (across files).
    [n          Jump to the previous note (across files).
    s           Toggle between unified and side-by-side split view.
    T           Cycle between full stack view and per-commit filtering.
    f           Focus the Files section in the explorer.
    t           Focus the Threads section in the explorer.
    q           Close the full review layout.

Notes list (N):~
                                                         *review-notes-list*

    j / n / ]n  Jump to next note entry.
    k / [n      Jump to previous note entry.
    <CR>        Jump to the note in the diff view.
    dd          Delete the note.
    s           Toggle the note between draft and staged.
    P           Publish all staged notes.
    y           Copy local notes, open threads, and discussion.
    Y           Copy only local draft/staged notes.
    C           Clear all local notes after confirmation.
    gd          Jump to a referenced note (`#<id>` in the note body).
    ?           Open review help.
    q / <Esc>   Close the list.

Note float:~

    <C-s>       Save the note and close.
    :w          Save the note and close.
    :wq         Save the note and close.
    <Esc>       Cancel and close (normal mode).

6. NOTES WORKFLOW                                              *review-notes*

Notes go through three stages:

  Draft (`-`)     Newly created notes start here.
  Staged (`+`)    Press `s` in the notes list to mark as ready to publish.
  Published (`*`) Press `P` to publish all staged notes at once.

Notes are persisted automatically across sessions. They are stored as JSON
in `vim.fn.stdpath("data")/review/`.

                                                        *review-notes-refs*
Note references~

You can reference other notes by typing `#<id>` in the note body (e.g.
"see #1 for context"). In the notes list:

- References are highlighted with a distinct color.
- Press `gd` on a line containing a reference to jump to that note.

Each note is assigned an auto-incrementing ID visible in the notes list
(e.g. `#1`, `#2`).

                                                    *review-notes-explorer*
Explorer threads~

The explorer keeps `Files` and `Threads` separate. Remote threads and local
notes are grouped under source labels such as `github/`, `gitlab/`, and
`local/`, and selecting a thread entry jumps to the related diff location.

7. CONFIGURATION                                      *review-configuration*

Default configuration: >lua

    require("review").setup({
      view = "unified",       -- "unified" or "split"
      render = {
        word_diff = {
          enabled = true,         -- inline word diff for paired changes
          max_line_length = 300,  -- skip very long lines
          max_pairs_per_hunk = 64,
          max_hunk_lines = 200,
          max_file_lines = 1500,  -- large files use line-level highlighting only
        },
      },
      notifications = {
        context = false,      -- suppress PR/base context info messages
      },
      colorblind = true,      -- blue/yellow scheme (default)
                              -- set to false for red/green
      keymaps = {
        add_note = "a",
        edit_note = "e",
        delete_note = "d",
        help = "?",
        next_hunk = "]c",
        prev_hunk = "[c",
        next_file = "]f",
        prev_file = "[f",
        next_note_short = "n",
        next_note = "]n",
        prev_note = "[n",
        toggle_split = "s",
        toggle_stack = "T",
        focus_files = "f",
        focus_threads = "t",
        notes_list = "N",
        suggestion = "S",
        close = "q",
      },
    })
<

==============================================================================
8. CHECKHEALTH                                          *review-checkhealth*

Run `:checkhealth review` to verify your environment:

- git is installed
- gh and/or glab CLI are installed
- gh/glab authentication is configured (`gh auth login`, `glab auth login`)
- Missing CLI is reported as an error for the detected forge
- Current directory is a git repository
